# Как правильно сдавать работы

1. Сделайте `git clone` репозитория себе на компьютер.
2. **Создайте новую ветку** в формате: `<номер группы>/Фамилия`  
   Пример: `23ПИ/ivanov`
3. В корне проекта создайте папку `<номер группы>/Фамилия`  
   Пример: `23ПИ/ivanov`
4. Внутри этой папки:
   - Для **основных заданий практики каждой пары** используйте подпапку `Practic`
   - Для **дополнительных заданий** — подпапку `Extra`
   - Пример структуры:
     ```
     23ПИ/
       ivanov/
         Practic/
           1_kub_intro/
           2_kub_deployment/
           ...
         Extra/
           bonus/
     ```
5. После завершения работы:
   - Сделайте `git add .`, затем `git commit -m "<краткое описание задачи>"`
   - Отправьте изменения командой `git push origin <ваша ветка>`
6. Оформите **pull request** в ветку `main` репозитория.
7. **Работа считается сданной только после отправки pull request в main.**

---

**Важно:**  
- Следуйте структуре папок, это снизит путаницу и ускорит проверку.
- Не складывайте все задания в одну папку!
- Один pull request = одна законченная часть работы (можно через доп. коммиты обновлять PR).
